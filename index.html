    WebSocket 기반 동시 편집 가능한 이음보드 구현

  • 좌석 배치 및 근태 관리

💡 기술적 도전과 해결

⚡ 비동기 처리와 높은 동시성

문제: 다수의 사용자가 동시에 알림을 받고 화이트보드를 사용할 때 서버 부하 증가

해결: Spring WebFlux와 Reactive Programming을 활용하여 비동기 논블로킹 I/O 처리 구현. 이를 통해 서버 자원을 효율적으로 사용하고 높은 동시성을 유지하면서도 응답 속도를 개선했습니다.

🔔 실시간 알림 시스템

문제: 다양한 외부 API(JIRA, Mattermost, GitLab)의 알림을 실시간으로 통합 제공 필요

해결: SSE(Server-Sent Events)를 활용하여 서버에서 클라이언트로 실시간 푸시 알림 구현. WebSocket보다 가벼운 SSE를 선택하여 단방향 알림에 최적화했습니다.

  • SSE를 통한 실시간 알림 시스템 구축으로 사용자 경험 향상

💡 기술적 도전과 해결

🔌 이기종 기기 호환성 확보

문제: 제조사별 기기 스펙 차이로 단일 제어 API 설계가 어려움

해결: 기기 속성을 추상화한 디바이스 프로필과 상태 모델을 정의하고, WebFlux 핸들러에서 공통 포맷으로 변환하는 어댑터 계층을 구현했습니다.
